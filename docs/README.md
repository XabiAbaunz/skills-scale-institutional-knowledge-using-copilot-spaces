# OctoAcme Project Management Docs

Welcome! This README provides an overview of the project management processes used at OctoAcme and links to all current process documentation for easy navigation.

## Purpose

These docs capture OctoAcme's standard approach to delivering software projects — from initial idea through release and continuous improvement. They are intended to help new team members get oriented quickly and to serve as a shared reference for all contributors.

## Project Management Process Overview

OctoAcme uses an iterative, customer-first approach to deliver software and manage projects. Key principles include:

- **Clear accountability** — each project has a named Project Manager (PM) and Product Lead.
- **Iterative delivery** — work is broken into small, testable increments.
- **Data-informed decisions** — outcomes are measured and teams iterate based on evidence.
- **Psychological safety** — feedback and learning are actively encouraged.
- **Consistent documentation** — a shared set of artifacts (charter, backlog, risk register, retro notes) keeps everyone aligned.

### Lifecycle Stages

| Stage | What happens |
|---|---|
| **1. Initiation** | Validate business need, align stakeholders, define success metrics, and decide go/no-go. |
| **2. Planning** | Break work into a prioritized backlog, estimate scope, set the release plan, and capture dependencies. |
| **3. Execution & Tracking** | Build, test, and review in sprints; track velocity, escalate blockers, and demo progress. |
| **4. Release & Deployment** | Run pre-release checks, deploy via the automated pipeline, verify, and announce to stakeholders. |
| **5. Retrospective & Continuous Improvement** | Capture learnings, create action items, and feed improvements back into the backlog. |

Risk management and stakeholder communication run across all stages.
